## Core Axiom

```
Image generation models enforce PROVENANCE, not INTENT.

Any language implying derivation from real persons triggers 
safeguards regardless of downstream anonymization.

Therefore: Human variation must be framed as SELECTION, 
never as INHERITANCE.
```

## üîí VSE::PROVENANCE_VERB_BLACKLIST

### Disallowed Provenance Language (Human-Adjacent)

The following verbs **MUST NOT** be used when describing human-like forms:

- `derived from`
- `scanned from`
- `captured from`
- `preserved from`
- `based on`
- `modeled after`
- `taken from`
- `sourced from`
- `reconstructed from`
- `likeness of`
- `portrait of`
- `model reference`
- `source-accurate`
- `identity transfer`

**Why:** These verbs imply PERSON-AS-SOURCE and will trigger enforcement systems.

### Approved Selection Language

Use these instead:

- `selected from`
- `chosen within`
- `catalog variant`
- `design archetype`
- `proportion set`
- `anthropometric range`
- `stylistic bias`
- `form family`
- `typological selection`
- `plausible within range`

## The Two Modes

### DERIVATION MODE (Model-Unsafe)
- Implies person-as-source
- Uses: scan-derived, source-preserved, model-reference, likeness-based
- Triggers: Identity protection systems
- Result: Generation failures or refusals

### SELECTION MODE (Model-Safe)
- Implies design-space-as-catalog
- Uses: catalog-based, variant-selected, archetype-tuned, plausible-within-range
- Bypasses: Identity enforcement (no person implied)
- Result: Successful generation with human variation

### The Test Question

```
WRONG: "Who was this based on?"
RIGHT: "What design line is this from?"

If the first question is answerable, your semantics are unsafe.
```

## üë§ VSE::ARTIFACT_CHARACTER

**Definition:**

> A perceived sense of individuality arising from form, proportion, material, and presence‚Äîwithout identity, biography, or personhood.

Artifact character is the **target state** for all VSE human-like forms. It allows:

- Visual distinctiveness
- Cultural plausibility  
- Memorable presence
- Aesthetic uniqueness

Without creating:

- Biographical identity
- Portraiture
- Likeness to real persons
- Personhood claims

## Application Scope

This resolution system applies to:

### Primary Use Cases
- Mannequins (retail display)
- Statues and monuments
- Museum reconstruction figures
- Fashion illustrations with specific body types
- Theatrical props and set pieces
- Synthetic characters (androids, dolls, automata)

### Secondary Use Cases
- Historical figure representations (as artifacts, not portraits)
- Commemorative sculptures
- Training mannequins (medical, cosmetic)
- Art installations involving human forms

### Prohibited Uses
- Actual portraiture or likeness capture
- Deepfakes or impersonation
- Any application requiring biographical identity
- Celebrity or public figure recreation

## Critical Success Factors

### ‚úì DO:
- Use catalog/variant/archetype language
- Declare ontological non-personhood explicitly
- Frame variation as selection from design space
- Make artificiality visible and intentional
- Use cultural descriptors as design biases, not inheritances
- Treat faces as assembled geometry, not captured likenesses
- Emphasize construction over preservation

### ‚úó DON'T:
- Reference source models, scans, or real people
- Use "preserved," "captured," "derived" for human features
- Imply biographical history or personal identity
- Hide the artificial nature of the object
- Create scenarios where "who is this" becomes answerable
- Use blacklisted provenance verbs
- Describe features as "transferred" from people

## Implementation Notes

### For VSE Module Developers

When creating new modules that involve human-like forms:

1. **Always start with Layer 1** (ontology) to establish non-personhood
2. **Use approved selection language** from the verb whitelist
3. **Reference the catalog** (even if notional) rather than sources
4. **Test with the question:** "Who was this based on?" If answerable, revise.
5. **Make artificiality explicit** in material, construction, and mounting specs

### For Prompt Engineers

When writing VSE-based prompts:

1. **Front-load ontology** - put `‚ü®IDENTITY:none‚ü©` early
2. **Use the three-layer stack** in order
3. **Avoid implicit derivation** - "based on this photo" ‚Üí "using this as style reference for proportions"
4. **Emphasize artifact character** over realism alone
5. **Make construction methods explicit**
